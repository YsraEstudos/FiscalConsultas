## 2024-04-30 - O(1) Sets and Single-Pass Regex Replace for ID Marker Injection
**Learning:** When injecting classes into HTML based on a list of IDs (e.g., `inject_comment_marks`), replacing an `O(N * K)` approach (looping through K keys and doing a full-text regex replacement over N length string) with an `O(N + K)` approach (doing a single-pass regex to find all elements with IDs, checking against an O(1) set of targets, and replacing if matched) can yield massive performance improvements (e.g. 4.1s down to 0.03s for large strings), especially when processing large rendered chapters. In addition, pre-compiling regular expressions at the module level avoids repetitive compilation overhead.
**Action:** When searching and replacing multiple specific targets in a large string, use a single-pass regex replacement function paired with O(1) sets for checking target matching, instead of looping over targets and running a replace for each one. Always elevate `re.compile()` calls to the module level when used inside frequently called formatting functions.
