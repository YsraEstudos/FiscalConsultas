## 2025-04-24 - Module-Level LRU Cache for NLP Stemming
**Learning:** Applying `@functools.lru_cache` on string processing logic (like NLP stemming) must be done on a module-level function or a `@staticmethod` rather than an instance method. When applied to an instance method, `self` becomes part of the cache key, causing complete cache misses across different instances and potential memory leaks. Furthermore, in text processing loops with large payloads (like NCM expansions where terms like "veiculos", "motores", "animais" repeat often), memoizing the pure stemmer output prevents redundant multi-step string manipulations (lowercasing, accent removal, suffix matching).
**Action:** Always extract inner, pure string operations to module-level functions before applying `lru_cache`, and ensure a reasonable `maxsize` to prevent unbounded memory growth.
